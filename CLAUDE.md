## About This Theme

This is a custom WordPress theme named **sv-homes-theme** built on [Sage 11](https://roots.io/sage/) by Roots. It uses:
- **Laravel Blade** templating via the [Acorn](https://github.com/roots/acorn) framework
- **Tailwind CSS v4** for styling
- **Vite** for asset bundling
- **PHP 8.2+** with PSR-4 autoloading under the `App\` namespace

## Development Commands

```bash
# Install PHP dependencies
composer install

# Install JS dependencies
npm install

# Start Vite dev server (hot module replacement)
npm run dev

# Build production assets
npm run build

# Lint PHP with Laravel Pint
./vendor/bin/pint

# i18n / translation workflow
npm run translate      # generate .pot and update .po files
npm run translate:compile  # compile .mo and .json files
```

The `APP_URL` environment variable is needed for the Vite plugin. It defaults to `http://example.test` if not set.

## Architecture

### PHP Side (`app/`)

- `functions.php` — Theme bootstrap: loads Composer autoloader, boots Acorn with `ThemeServiceProvider`, and includes `app/setup.php` and `app/filters.php`.
- `app/setup.php` — WordPress theme setup: nav menus, theme supports, widget sidebars, block editor styles/scripts.
- `app/filters.php` — WordPress filters (e.g., excerpt customization).
- `app/Providers/ThemeServiceProvider.php` — Laravel service provider extending `SageServiceProvider`. Register custom services or bindings here.
- `app/View/Composers/` — Blade view composers that inject data into templates. `App.php` applies to all views and provides `$siteName`. `Post.php` and `Comments.php` provide post/comment data.

### Frontend (`resources/`)

- `resources/css/app.css` — Tailwind CSS entry point. Uses `@source` directives to scan `views/` and `app/` for utility classes.
- `resources/css/editor.css` — Block editor styles.
- `resources/js/app.js` / `editor.js` — JS entry points.
- `resources/views/` — Blade templates:
  - `layouts/app.blade.php` — Root HTML layout; includes header/footer sections and yields `content` and `sidebar`.
  - `sections/` — Full-page sections: `header.blade.php`, `footer.blade.php`, `sidebar.blade.php`.
  - `partials/` — Reusable content fragments (post content, entry-meta, page-header, etc.).
  - `components/` — Blade components (e.g., `alert.blade.php`).
  - `forms/` — Form templates.
  - Top-level template files map to WordPress template hierarchy: `index.blade.php`, `single.blade.php`, `page.blade.php`, `404.blade.php`, `search.blade.php`.

### Build Output (`public/build/`)

Generated by Vite — do not edit manually. The `wordpressThemeJson` Vite plugin generates `public/build/assets/theme.json` from Tailwind config, which WordPress uses for block editor settings (overriding the root `theme.json`).

## Key Conventions

- **Styling — use Tailwind**: Prefer Tailwind utility classes over inline `style=""` or new custom CSS. Theme colors/fonts are in `resources/css/app.css` under `@theme` (e.g. `--color-sv-navy`, `--font-display`), which generate utilities like `bg-sv-navy`, `text-sv-gold-light`, `font-display`. Use the shared classes `.sv-page-hero`, `.sv-container`, `.sv-section`, etc. for layout; use Tailwind for spacing, typography, and one-off styling.
- **View data**: Add data to views by creating or extending a Composer in `app/View/Composers/`. Composers extend `Roots\Acorn\View\Composer` and declare which views they apply to via `protected static $views`.
- **WordPress hooks**: Add theme actions/filters in `app/setup.php` (setup-related) or `app/filters.php` (filter-related), not in `functions.php`.
- **Tailwind classes**: Tailwind v4 scans `resources/views/` and `app/` automatically — no need to update a content config array.
- **Text domain**: `sage` is the i18n text domain used throughout.
